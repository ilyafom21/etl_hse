# ETL Pipeline (MongoDB → PostgreSQL)

Проект выполнен в рамках итогового задания по модулю **ETL-процессы**.

Реализован полноценный ETL-конвейер, в котором данные из MongoDB извлекаются, преобразуются и загружаются в PostgreSQL, после чего используются для построения аналитических витрин.

---

## Общая логика решения

Пайплайн устроен следующим образом:

MongoDB → Airflow → PostgreSQL → витрины

Вся обработка данных централизована в Airflow, который отвечает за выполнение и управление задачами.

---

## Используемые технологии

- MongoDB — источник данных

- Apache Airflow — управление пайплайнами  
- Python — реализация ETL-логики

## Источник данных

В MongoDB генерируются тестовые данные, моделирующие работу системы:

- пользовательские сессии  
- события пользователей  
- обращения в поддержку  

Данные создаются с помощью скрипта:

python scripts/generate_mongo_data.py

---

## ETL-процесс

Основной ETL реализован в Airflow в виде DAG.

В рамках процесса выполняется:

- извлечение данных из MongoDB  
- преобразование структуры данных  
- загрузка в PostgreSQL  

### Трансформация данных

MongoDB содержит вложенные структуры и массивы, которые были приведены к реляционному виду:

- pages_visited → отдельная таблица session_pages  
- actions → отдельная таблица session_actions  
- messages → таблица ticket_messages  

Это позволило нормализовать данные и упростить дальнейшую аналитику.

### Особенности загрузки

- реализована идемпотентность  
- при повторном запуске данные не дублируются  
- используется подход с обновлением существующих записей  

---

## Хранилище данных (PostgreSQL)

Основные таблицы:
- user_sessions  
- event_logs  
- support_tickets  

Связанные таблицы:
- session_pages  
- session_actions  
- ticket_messages  


## Аналитический слой

После загрузки данных формируются витрины (схема `mart`).

### user_activity

Позволяет анализировать поведение пользователей:

- количество сессий  
- среднюю длительность  
- количество действий  
- количество посещённых страниц  

Пример запроса:
SELECT * FROM mart.user_activity;

---

### support_stats

Отражает работу службы поддержки:

- количество тикетов  
- статус обращений  
- типы проблем  
- среднее время обработки  

Пример запроса:
SELECT * FROM mart.support_stats;

---

## Оркестрация (Airflow)

В проекте реализованы два DAG:

### 1. mongo_to_postgres_dag
Выполняет ETL:
- извлекает данные из MongoDB  
- преобразует  
- загружает в PostgreSQL  

### 2. marts_dag
Строит аналитические витрины:
- пересчитывает агрегаты  
- обновляет таблицы витрин  

---

## Запуск проекта

1. Поднять окружение:

docker compose up -d

2. Сгенерировать данные:

python scripts/generate_mongo_data.py

3. Открыть Airflow:

http://localhost:8080

4. Запустить DAG:
- mongo_to_postgres_dag  
- marts_dag
